Giao thức HTTP (Hyper Text Transfer Protocol)

HTTP (HyperText Transfer Protocol - Giao thức truyền tải siêu văn bản) là giao thức nằm ở tầng Application Layer, được sử dụng để liên hệ thông tin giữa các máy chủ dịch vụ và máy sử dụng dịch vụ.

1. Giới thiệu

Từ lúc ra đời đến nay HTTP trải qua nhiều phiên bản, tuy nhiên phiên bản hoàn chỉnh đầu tiên của HTTP là HTTP 0.9 (năm 1991), tiếp theo là HTTP 1.0 (năm 1996), HTTP 1.1 (năm 1997) và sau đó gần đây nhất là 2x (HTTP 2.0).

Tuy phiên bản 2.0 đã ra đời nhưng vì một số lý do mà nó hiện vẫn chưa được phổ biến. Phiên bản HTTP 1.1 hiện vẫn là phiên bản HTTP phổ biến nhất, có thể tham khảo thêm tại RFC 2616. Sự khác biệt lớn giữa HTTP/1.0 và HTTP/1.1 là HTTP/1.0 sử dụng một kết nối mới cho mỗi Yêu cầu/Phản hồi trao đổi, trong khi đó kết nối trong HTTP/1.1 có thể được sử dụng cho một hoặc nhiều Request/Response (Yêu cầu/Phản hồi).

Một số regex hay dùng

Regex (Regular Expressions) hay biểu thức chính quy là một tập các ký tự dùng để mô tả một chuỗi nào đó theo những quy tắc, cú pháp nhất định. Regex không những được sử dụng trong lập trình mà còn kết hợp mạnh mẽ đối với các công cụ như grep, sed,… trên môi trường Linux.

Một số ký tự cơ bản trong regex:

Ký tự Ý nghĩa
\ Đươc sử dụng để bỏ qua ký tự đặc biệt
^ Ký tự đánh dấu bắt đầu 1 chuỗi
$ Ký tự đánh dấu kết thúc 1 chuỗi
. Một ký tự (đơn) bất kỳ
| sử dụng như OR. Ví dụ IPv4 | IPv6
() Sử dụng như GROUP (độ ưu tiên). Ví dụ IP (v4 | v6)
[] Một dãy các ký tự. Ví dụ [abc]. Có thể kết hợp với ^ ở đầu để chỉ phủ định (NOT) hoặc với - để chỉ ký tự liên tiếp như [0-9] (từ 0 -> 9) hay [a-z] (từ a -> z)
? Kiểm tra có 0 hoặc 1 ký tự phía trước. Ví dụ https? sẽ khớp với http và https
* Biểu diễn 0 hoặc 1 hoặc nhiều hơn 1 ký tự bất kỳ phía trước
+ Ký tự phía trước xuất hiện 1 hoặc nhiều lần
{n} Ký tự phía trước xuất hiện n lần
{min,} Ký tự phía trước xuất hiện min lần trở lên
{min,max} Ký tự phía trước xuất hiện tối thiểu min lần và tối đa max lần

Một số cái beep trong shell

Hôm nay tôi lang thang và thấy mấy cái này và tôi biết tôi cần note lại cho mình. Tôi cũng xin nói luôn tôi cũng chưa biết gọi chúng là cái gì nên tạm thời gọi là cái beep vậy. Khi làm việc với shell trên linux và… có đôi chút về bash thì nó thật sự hữu ích :smiley:

Command Description Example
& Chạy lệnh với background ls &
&& Biểu thức AND if [ “$foo” -ge “0” ] && [ “$foo” -le “9”]
** Biểu thức OR if [ “$foo” -lt “0” ] ** [ “$foo” -gt “9” ]
^ Ký tự bắt đầu dòng/chuỗi grep “^foo”
$ Ký tự kết thúc dòng/chuỗi grep “foo$”
= Phép bằng hoặc gán if [ “$foo” = “bar” ]
! Phép NOT if [ “$foo” != “bar” ]
$$ PID của shell hiện tại echo “my PID = $$”
$! PID của background command cuối cùng ls & echo “PID of ls = $!”
$? Trạng thái thoát của lệnh cuối cùng ls ; echo “ls returned code $?”
$0 Tên của lệnh hiện tại echo “I am $0”
$1 Đối số đầu tiên của lệnh hiện tại echo “My first argument is $1”
$9 Đối số thứ 9 của lệnh hiện tại echo “My ninth argument is $9”
$@ Tất cả các đối số (parameters) của lệnh hiện tại echo “My arguments are $@”
$* Tất cả các đối số của lệnh hiện tại (trừ khoản trống và trích dẫn) echo “My arguments are $*”
-eq So sánh bằng if [ “$foo” -eq “9” ]
-ne So sánh không bằng if [ “$foo” -ne “9” ]
-lt So sánh nhỏ hơn if [ “$foo” -lt “9” ]
-le So sánh nhỏ hơn hoặc bằng if [ “$foo” -le “9” ]
-gt So sánh lớn hơn if [ “$foo” -gt “9” ]
-ge So sánh lớn hơn hoặc bằng if [ “$foo” -ge “9” ]
-z Chuỗi có chiều dài 0 if [ -z “$foo” ]
-n Chuỗi có chiều dài khác 0 if [ -n “$foo” ]
-d Là một thư mục if [ -d /bin ]
-f Là một file if [ -f /bin/ls ]
-r Là một file có thể đọc được if [ -r /bin/ls ]
-w Là một file có thể ghi được if [ -w /bin/ls ]
-x Là một file có thể thực thi if [ -x /bin/ls ]

Cài đặt Arch Linux trên Virtualbox [Notes]

Arch Linux là một hệ điều hành họ Linux. Đây không phải là một hệ điều hành cho người mới bắt đầu. Bạn sẽ phải định dạng, install tất cả bằng tay khi cài đặt. Trong hướng dẫn này chúng ta sẽ xem cách cài đặt. Có thể nó sẽ không khó như bạn đã nghĩ.

1. Giới thiệu

Chúng ta sẽ xét qua một vài điểm xem như là ưu điểm của distro này sau đây.

  • Đối với Ubuntu, Fedora, Linux mint,… sau khi cài đặt chúng ta sẽ được tự động cài đặt một số phần mềm đi kèm như LibreOffice, GIMP,… Tuy nhiên với Arch, bạn sẽ không có gì và tất nhiên bạn phải tự tìm và cài đặt những thứ mình cần. Có lẽ bạn nghĩ sẽ thật phức tạp, tuy nhiên chúng ta cũng xét qua một khía cạnh khác rằng nó sẽ nhẹ hơn, nhanh hơn, tùy chỉnh hơn với người dùng.

  • Với Arch bạn sẽ có AUR (Arch user repository) - đây là kho dự trữ của Arch, nó tuyệt vời hơn bạn nghĩ. Nó cung cấp cho bạn rất nhiều các gói phần mềm không có sẵn trong kho chính thức.

  • Trình quản lý pacman tự động biên dịch các gói cho bạn.

  • Rolling release: Điều này có nghĩa là bạn sẽ luôn có những bản mới nhất, các gói, kenel, tất cả chỉ đợi bạn trải nghiệm

  • Cuối cùng giờ đây Arch cũng dần có lượng cộng đồng người dùng đông đảo, và Wiki Arch cũng hỗ trợ đầy đủ và hiện thời rất tốt.

Với Arch việc cài đặt như đã nói sẽ có chút phức tạp hơn các distro khác. Tuy nhiên cũng có những dự án nhằm đơn giản hóa cài đặt Arch như Manjaro, Antergos, Ninja và nhiều thứ khác. Bạn có thể tải và cài đặt.
